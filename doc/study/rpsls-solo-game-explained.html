Reglas declarativas, no una cadena de if

Cada relación se representa como datos. Esto permite mostrar las mismas reglas en pantalla y convertirlas en reglas ejecutables sin duplicar conocimiento.

2. Conversión a reglas ejecutables

frontend/lib/rpsls.ts · Engine
const rules = WINNING_RELATIONS.map((relation) => ({
  conditions: {
    all: [
      { fact: 'first', operator: 'equal', value: relation.winner },
      { fact: 'second', operator: 'equal', value: relation.loser },
    ],
  },
  event: { type: 'winning-relation', params: relation },
}))

const engine = new Engine(rules)

json-rules-engine recibe hechos, prueba las condiciones y emite el evento de la combinación coincidente. Así, el algoritmo no depende de diez bloques condicionales escritos manualmente.

04 / CICLO DE PARTIDA

Qué ocurre al pulsar una jugada

La función de ronda coordina interacción, aleatoriedad, motor de reglas y actualización inmutable del estado.

frontend/app/games/page.tsx · playRound
const playRound = async (choice: RpslsChoice) => {
  if (playingRef.current || matchWinner) return
  playingRef.current = true
  setIsPlaying(true)

  const computer = randomRpslsChoice()
  await new Promise((resolve) => setTimeout(resolve, 550))
  const resolution = await resolveRpslsRound(choice, computer)

  setGame((current) => ({
    ...current,
    playerScore: current.playerScore + (resolution.result === 'win' ? 1 : 0),
    history: [nextRound, ...current.history].slice(0, 12),
  }))
}
Guard clause

Impide jugar durante otra ronda o después de finalizar la partida.

550 ms

Hace visible el estado “La CPU está eligiendo” sin alterar las reglas.

setGame(current)

Usa siempre el estado más reciente, importante ante actualizaciones asíncronas.

slice(0, 12)

Limita la memoria persistida aunque la partida libre sea muy larga.

05 / PERSISTENCIA

La partida sobrevive a una recarga

El juego usa localStorage, una base clave-valor del navegador. No necesita una tabla en PostgreSQL porque no comparte la partida con otros usuarios.

frontend/app/games/page.tsx · hidratación
useEffect(() => {
  try {
    const saved = window.localStorage.getItem(STORAGE_KEY)
    if (saved) setGame(normalizeSavedGame(JSON.parse(saved)))
  } catch {
    window.localStorage.removeItem(STORAGE_KEY)
  }
  setHasHydrated(true)
}, [])

useEffect(() => {
  if (hasHydrated) localStorage.setItem(STORAGE_KEY, JSON.stringify(game))
}, [game, hasHydrated])
No se confía ciegamente en el navegador

normalizeSavedGame verifica tipos, valores positivos, modos permitidos y nombres de jugadas. Si el JSON está roto, se elimina. Así se evita el error anterior de intentar leer label de una jugada inexistente.

07 / DJANGO

Por qué hacía falta una migración de unión

Dos ramas de trabajo crearon migraciones con el mismo número 0016 y dependencias distintas. Ambas son válidas, pero Django necesita un único punto final para saber el orden del historial.

→
0017_merge_online_status_projects_evaluations
nuevo nodo final común
0016_merge_projects_evaluations
une proyectos y evaluaciones
→
operations = []
no modifica tablas ni datos
backend/sync/migrations/0017_merge_online_status_projects_evaluations.py
from django.db import migrations

class Migration(migrations.Migration):
  dependencies = [
    ('sync', '0016_campususer_last_active_time'),
    ('sync', '0016_merge_projects_evaluations'),
  ]
  operations = []
Una migración de merge no es un merge de Git

Git combina archivos. Esta migración combina dos caminos del grafo histórico de Django. operations = [] significa que no ejecuta SQL adicional.

09 / DEFENSA

Cómo explicar esta implementación

Una defensa clara empieza por el alcance, sigue el recorrido de una jugada y termina con decisiones técnicas y pruebas.

Resumen de 60 segundos

“Añadimos un juego individual RPSLS en la ruta /games. Las diez relaciones ganadoras están declaradas una sola vez y json-rules-engine las evalúa. React controla la ronda, la puntuación y la revancha; localStorage conserva la partida y una función normalizadora protege frente a datos inválidos. La CPU usa crypto.getRandomValues. La interfaz es responsive y accesible. Además, una migración vacía une dos ramas del grafo de Django, y make test funciona sin interacción. No hay chat ni multijugador en esta rama.”

App Router
Sistema de rutas de Next.js basado en carpetas dentro de app.
Hook
Función de React que añade estado, efectos o referencias a un componente.
Hidratación
Momento en que React activa en el navegador el HTML generado.
localStorage
Almacenamiento persistente del navegador asociado al origen.
Motor de reglas
Evalúa hechos contra condiciones declaradas y emite eventos.
Migración merge
Nodo que reúne varias ramas del grafo de migraciones Django.
Estado inmutable
Se crea un objeto nuevo en vez de modificar directamente el anterior.
Guard clause
Retorno temprano que impide ejecutar una operación inválida.
Responsive
Diseño que reorganiza su estructura según el espacio disponible.

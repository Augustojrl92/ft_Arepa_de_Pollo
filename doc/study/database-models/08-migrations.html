08. Migraciones y evolución del esquema

Qué son las migraciones, dónde están, cómo se aplican y qué cuentan sobre la historia del modelo de datos.

🧩 Django ORM 🐘 PostgreSQL 🌐 API 42 🔄 Sync local 📚 Referencias al repo

Qué son migraciones

Son archivos Python versionados que describen cambios del esquema SQL: crear tablas, añadir campos, borrar modelos y crear índices.

Dónde están

  • backend/sync/migrations/
  • backend/users/migrations/
  • backend/authentication/migrations/
  • backend/coalitions/migrations/

Línea de evolución del esquema

Estas son las migraciones que importan para entender por qué el modelo hoy es como es.

0005 sync · nace Coalition

backend/sync/migrations/0005_coalition.py

La coalición persistente se mueve al app sync.

0010 sync · nacen snapshots

backend/sync/migrations/0010_coalitionscoresnapshot_campususerscoresnapshot.py

Se añade histórico diario para coaliciones y usuarios.

0012 sync · nace SyncMetadata

backend/sync/migrations/0012_syncmetadata.py

Se añade observabilidad del sync.

0014 sync · nace project cursor

backend/sync/migrations/0014_coalitionprojectcursor.py

Empieza la capa incremental de proyectos.

0015 sync · nace evaluation cursor

backend/sync/migrations/0015_campususer_coalitions_user_id_and_cursor.py

Se añade coalitions_user_id y el cursor de evaluaciones.

0011 authentication · desaparece FortyTwoProfile

backend/authentication/migrations/0011_delete_fortytwoprofile.py

La auth propia histórica se simplifica hacia User builtin.

0003 coalitions · desaparece Coalition antiguo

backend/coalitions/migrations/0003_delete_coalition.py

La persistencia real de coaliciones ya no vive en esa app.

0001 users · nace FriendsList

backend/users/migrations/0001_initial.py

Arranca la capa social local.

0002 users · nace UserPreferences

backend/users/migrations/0002_userpreferences.py

Arranca la capa de UX persistida.

Errores que explican las migraciones

Por qué esta parte importa de verdad.

  • Tabla o columna inexistente: el código espera un campo nuevo, pero la migración no se aplicó.
  • Modelo histórico confuso: leer apps vacías sin mirar migraciones lleva a conclusiones equivocadas.
  • Sync roto tras deploy: el código usa cursores, snapshots o metadatos que la BD aún no tiene.

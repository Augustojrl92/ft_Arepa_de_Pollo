User vs CampusUser

La confusion mas comun es pensar que ambos modelos hacen lo mismo. No es asi: User resuelve auth; CampusUser resuelve el dominio.

🔐

User

  • modelo builtin de Django
  • tabla auth_user
  • sesion, permisos, cookies/JWT
  • owner de preferencias y amigos
🛰

CampusUser

  • modelo en backend/sync/models.py
  • tabla sync_campususer
  • login, intra_id, level, scores, metrics
  • centro del dominio del producto

Relacion OneToOne

El enlace es opcional desde CampusUser hacia User. Un usuario autenticado puede asociarse con un solo perfil de campus.

En codigo real, CampusUser define django_user = models.OneToOneField(User, ...) con related_name="campus_user_profile".

Campos importantes

Lo que mas conviene memorizar es que User guarda identidad interna y CampusUser guarda negocio sincronizado.

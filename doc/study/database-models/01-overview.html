01. Visión global del sistema

Mapa general de modelos, fuente de verdad local, ERD simplificado, flujo de datos y pseudocódigo base.

🧩 Django ORM 🐘 PostgreSQL 🌐 API 42 🔄 Sync local 📚 Referencias al repo

Idea central

La app no se apoya en consultas en vivo a 42 para pintar cada pantalla. Primero sincroniza, persiste en PostgreSQL local y luego sirve APIs internas que consumen las vistas.

User

Identidad interna de Django: autenticación, sesión y permisos.

CampusUser

Modelo central del dominio sincronizado desde 42.

Coalition

Estado actual local de cada coalición.

Snapshots

Fotos diarias para evolución histórica.

Cursores

Puntos de reanudación para sync incremental.

SyncMetadata

Último sync y salud operativa.

FriendsList / UserPreferences

Datos propios de producto, no de 42.

Lo importante no es solo qué tablas existen, sino cómo se alimentan y quién las consume.

Fuente de verdad local

Frontend Next.js y endpoints Django consultan la base local. La API 42 solo alimenta el proceso de sync.

Pseudocódigo global

Versión mental compacta del sistema completo.

FUNCIÓN modelo_de_datos_global():

    User = identidad interna Django
    CampusUser = dominio sincronizado desde 42
    Coalition = estado actual de coaliciones

    SI llega sync nuevo:
        actualizar CampusUser
        actualizar Coalition

    SI quiero histórico:
        crear snapshots diarios

    SI quiero incremental:
        guardar cursores por coalición

    SI quiero UX o capa social:
        usar UserPreferences y FriendsList

    devolver base local lista para endpoints
